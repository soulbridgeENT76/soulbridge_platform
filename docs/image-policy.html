SoulBridge CMS · 엔지니어링 레퍼런스

이미지 업로드 · 불러오기 정책

관리자 CMS가 이미지를 받고, 처리하고, 저장하고, 사이트에 표시하기까지의 규칙입니다. 대상별로 다르게 적용되는 크롭·리사이즈·검증 기준을 한곳에 정리했습니다.

저장 Supabase Storage · 버킷 images 표준 포맷 WebP (로고 SVG 예외) 렌더 next/image + plain <img>
01

공통 파이프라인

모든 이미지가 거치는 다섯 단계

01 · 선택

관리자 업로드

공용 AdminImageUpload로 파일 선택. 대상별 허용 포맷 제한.

02 · 처리

브라우저 변환 (선택 즉시)

파일 선택 순간 크롭·리사이즈·여백 트림·WebP를 클라이언트에서 수행하고 미리보기에 결과를 바로 반영. SVG 로고만 원본 통과.

03 · 업로드

Storage 저장

서버는 처리된 파일을 업로드만 — 이미지 가공은 하지 않음. 버킷 images/…, 쓰기 인증 필요.

04 · 기록

DB엔 경로만

전체 URL이 아닌 상대 경로 저장 (예: content/…​.webp).

05 · 표시

렌더 시 조립

mediaUrl()로 URL 생성 → next/image 또는 plain img.

업로드 시 폼은 파일과 함께 <name>_width <name>_height <name>_cleared 를 함께 제출합니다 — 인코딩된 치수와 "삭제됨/미변경" 구분을 서버에 전달하기 위함입니다.
02

공통 제약

대상과 무관하게 항상 적용

용량 · 포맷

  • 최대 파일 용량 30MB (클라이언트 사전 검사)
  • 저장 포맷은 WebP 로 통일 — 로고 SVG만 예외
  • SVG는 로고에서만 허용, 그 외 대상은 차단

스토리지 버킷 images

  • 허용 MIME image/webp · image/svg+xml
  • 파일 크기 상한 50MB, 공개 읽기 / 인증 쓰기
  • 폴더: content · artist · home · logo · about

저장값 규약

  • DB엔 상대 경로만 (URL·호스트 미포함)
  • 렌더 시 mediaUrl(path)로 절대 URL 조립
  • YouTube는 경로 대신 영상 id 저장

대상별 정책

업로드 시 처리와 크롭 시점이 대상마다 다름

아티스트 프로필 PNG · JPG · WebP WebP + 폭 ≤1200 축소만크롭·업스케일 없음 렌더3:4 유지 세로 3:4 · 권장 1200×1600

로고헤더·푸터·사이드바·히어로 SVG · PNG SVG 원본 그대로 / PNG는 WebP·높이 512·여백 자동 트림 —높이 기준 스케일 투명 필수 · 단색(검정) · PNG 세로 512px 권장
콘텐츠 · 아티스트 · ABOUT은 "크롭은 렌더에서" 정책을 씁니다 — 업로드는 원본 비율 그대로 저장(폭 상한만)하고, 표시할 때 object-cover로 각자의 비율(콘텐츠 카드 16:9, 아티스트·ABOUT 3:4)로 자릅니다. 콘텐츠 상세만 크롭 없이 원본 비율로 보여줍니다. 배너는 데스크톱·모바일이 서로 다른 크롭이라 업로드 시점 크롭을 유지합니다.
04

불러오기 · 렌더 정책

해상도는 next/image가, 비율은 CSS가 담당

next/image (최적화)

  • sizes로 뷰포트·DPR별 해상도 자동 제공
  • 목록 카드는 object-cover로 16:9 크롭
  • 아티스트·배너·ABOUT 등 대부분 이 경로

plain <img> (비최적화)

  • 콘텐츠 상세: 원본 비율 표시 (마스터 로드)
  • 로고: 벡터라 최적화 불필요 · SVG XSS 안전
  • 로고 높이 기준 h-9~h-12, 히어로 360→480px

YouTube 썸네일

  • img.youtube.com/vi/<id>/mqdefault.jpg
  • 저장은 영상 id, 카드엔 재생 아이콘 오버레이
  • 상세는 썸네일 대신 임베드 재생

허용 호스트 · 폴백

  • 원격 허용: Supabase 호스트 · img.youtube.com/vi/**
  • 이미지 없음 → PlaceholderImage (CSS)
  • 로고 없음 → 번들 /logo.png
05

검증 규칙

모두 클라이언트에서 확인 후 즉시 메시지 표시

콘텐츠 · 아티스트 · 배너 · ABOUT

  • 허용 어떤 크기든 업로드 가능
  • 권장 표시 해상도만 안내, 강제 없음
  • 반려 30MB 초과 파일

로고 · PNG

  • 반려 불투명 배경 · 전체 투명
  • 자동 주변 투명 여백 트림 (반려 아님)
  • 권장 세로 512px 이상

로고 · SVG

  • 허용 원본 그대로 저장, 크기 제한 없음
  • 반려 viewBox·크기를 읽을 수 없는 파일
  • 비율은 viewBox에서 추출해 저장
자동 · 허용 업로드 시 크롭 반려
06

핵심 원칙

정책이 서로 어긋나지 않게 하는 기준

저장은 WebP, DB엔 경로만 포맷과 호스트를 한곳에서 통제. 로고 SVG만 벡터 특성상 예외.
크롭과 해상도는 분리해서 생각 비율(크롭)은 CSS object-cover, 해상도는 next/image의 sizes가 담당. 콘텐츠·아티스트·ABOUT는 이 원칙을 그대로 따라 업로드 크롭을 뺐다.
작은 이미지를 억지로 키우지 않는다 콘텐츠·아티스트·ABOUT는 업스케일 없이 원본 비율을 유지(폭 상한만) — 흐려짐 방지.
가공은 전부 클라이언트, 서버는 저장만 크롭·리사이즈·여백 트림·WebP 변환은 파일 선택 시점의 브라우저에서 끝나고 미리보기에 즉시 반영된다. 서버 액션은 처리된 파일을 스토리지에 올릴 뿐 이미지 디코더가 없다.
업로드 검증은 UX, 최종 방어는 서버 클라이언트 검증은 즉시 피드백용. 서버 액션이 같은 규칙을 다시 확인한다.
